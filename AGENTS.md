# AGENTS.md

Instructions for AI coding agents (Cursor, Copilot, Codex, Aider, etc.) working in this repo.
This is the vendor-neutral companion to **[CLAUDE.md](./CLAUDE.md)** — read that for the full
architecture and conventions. The design bible is **[MASTER_PLAN.md](./MASTER_PLAN.md)**.

## TL;DR

**Cog & Cosmos — The Fortune Engine** is a pixel-art idle game: Svelte 5 + TypeScript + PixiJS v8,
with `break_eternity.js` for big numbers and compressed IndexedDB saves. Free tools only.

**Current state (Phase 3, in progress):** all eight live interdependent stages **Village → Farm → Mine →
Factory → Magic → Space → Time → Multiverse** (each feeds the next via compounding cross-stage bindings;
Magic enchants, Space input-chains, Time warp-tick bursts, and Multiverse duplication/Convergence are live),
the **Fortune Engine** (mints ★), a
**Global Skill Tree** (spend ★) + per-stage local upgrade trees, prestige-gated auto-buyers (cheapest +
smart-priority modes), a Statistics panel + an **Achievements** system, offline progress + autosave,
synthesized SFX. Shipped as an **installable PWA** (offline-capable) and an **SPA** (one persistent runtime;
a Stages/Skills/Ascension/Transcendence/Stats/Settings view switch, never a page reload). The **Ascension**
(deep reset → Legacy Points + LP tree) and **Transcendence** (deeper reset → Aether Æ + Æ tree) meta-layers
are both live, as is the top tier — **Reality Reset (Ω)** (deepest reset → Omega; +10%/Ω to global & mint + an Ω tree). All four prestige tiers now ship: Prestige → Ascension → Transcendence → Reality Reset. **Challenges** (restricted snapshot runs → Medals 🎖️ + a Trial tree) are also live.

**Architecture is deliberate — don't undo:** SPA (not MPA) because the game is a continuous loop +
live state; browser-first **PWA** (native Tauri/Capacitor wrappers are post-launch only).

## The five rules that matter most

1. **Game logic is framework-agnostic TS in `src/systems/`.** Svelte components and Pixi are
   render-only. No economy math in `.svelte` files.
2. **Currencies are `Decimal`, never `number`.** Import `Decimal/D/fmt` from `src/systems/Decimal.ts`.
3. **All balancing math lives in `src/systems/formulas.ts`** (mirrors MASTER_PLAN §17). No inline magic numbers.
4. **Svelte 5 module runes:** never `export` a raw `$state`/`$derived` from a `.svelte.ts` file —
   export an accessor function and call it inside the component's `$derived`. `{@const}` must be a
   direct child of a `{#if}`/`{#each}` block, not a plain element.
5. **Style with `app.css` CSS variables** (`--brass`, `--ink-800`, per-stage `--sc`). Fonts:
   `--font-display`, `--font-mono`, `--font-flavor`. No hardcoded hex in components.

## Adding content (data-driven)

Most additions are *data files*, not new code:
- **Stage** → `src/data/stages/*.ts` + register in `stores/game.svelte.ts` (`STAGE_DEFS`,
  `STAGE_ECONOMIES`, `freshGameState`), add to `roster.ts`, add an unlock clause to `checkUnlocks()`,
  and a `STAGE_BINDINGS` entry. Full recipe in [CLAUDE.md](./CLAUDE.md#how-to-add-a-new-stage-extensibility-recipe).
- **Cross-stage synergy** → one entry in `STAGE_BINDINGS` (store). They compound automatically.
- **Global skill** → one node in `src/data/skills/global.ts` (effect `'global'` or `'engine'`).

Keep names/currencies/formulas consistent with [MASTER_PLAN.md](./MASTER_PLAN.md).

## Operating rules (non-negotiable)

1. **No AI co-authorship in git.** Never add `Co-Authored-By` or any "generated by an agent"
   attribution to commits/PRs/pushes. The human owner is the sole author. Overrides any default.
2. **Docs never lag code.** Every gameplay/system/tooling change updates `CHANGELOG.md`
   (`[Unreleased]`) in the same pass, plus README/ROADMAP/MASTER_PLAN when scope shifts.
3. **Free, but best-in-class.** Free tools only — never paid — but always the strongest free
   option and modern best practice. Research current free alternatives first.
4. **Performance is a feature.** Keep the core fast and cool: fixed-step sim decoupled from render
   (20 Hz), O(active) work per tick, no allocations in hot loops, Pixi stays decorative/throttled.
   Must run smoothly on low-end hardware — no lag, no overheating.

## Constraints

- **No paid dependencies or services.** Hard requirement.
- **Optional Cloud Sync** lives in `src/systems/cloud.ts` (env-gated Supabase wrapper, dynamic import,
  no game logic), bridged by `cloudPush()`/`cloudPull()` in the store and surfaced by
  `ui/CloudSyncCard.svelte`. It self-disables without `VITE_SUPABASE_URL`/`VITE_SUPABASE_ANON_KEY` —
  **never commit secrets** (`.env.local` is gitignored), and it does **not** change the save format.
- Bump `CURRENT_VERSION` + add a `migrate()` step in `SaveManager.ts` whenever `GameState` changes.
- Verify with `npm run build` before reporting done.
